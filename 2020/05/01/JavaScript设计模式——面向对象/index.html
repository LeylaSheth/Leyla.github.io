JavaScript设计模式——面向对象

概念

类

类，即模板，如同es5中的构造函数

对象

根据类模板构造实例，new一个实例

三要素：继承、封装、多态

继承

子类继承父类

react开发中很常见

  • 父类是公用的
  • 继承可以将公共方法抽离出来，提高复用

封装

数据的权限和保密

  • public完全开放
  • protected对子类开放
  • private对自己开放

然而es6不支持。。。

TypeScript

new Student的实例中，friend无法访问到，直接报错，只能在模板中使用

  • 减少耦合，不该外露的不外露
  • 利于数据、接口的权限管理
  • ES6不支持，一般认为_开头的属性是private

多态

同一接口不同实现

  • 保持子类的开放性和灵活性
  • 面向接口编程
  • JS引用极少

面向对象——数据结构化
